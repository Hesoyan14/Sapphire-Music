# Sapphire: пошаговая инструкция с нуля (Windows)

Здесь описано **что именно открывать, куда нажимать и что вводить**, если вы почти не работали с Python и терминалом.

Если вам нужен только **API и загрузка файлов с другого компьютера**, после того как сервер у вас запущен, смотрите также [CLOUD-UPLOAD-RU.md](CLOUD-UPLOAD-RU.md).

---

## 1. Что это за программа

**Sapphire** — это музыкальный плеер с веб-интерфейсом, который работает **на вашем компьютере**. Он поднимает небольшой **сервер** (программу, которая отвечает на запросы в браузере) по адресу **`http://127.0.0.1:5000`**.

- **`127.0.0.1`** — это «сам этот компьютер», не интернет.
- **`5000`** — номер **порта** (как «дверь», через которую заходит браузер).

Пока сервер **запущён**, вы можете открыть сайт в браузере. Если сервер **закрыли** — страница не откроется.

---

## 2. Что должно быть установлено

Нужен **Python 3** (версия 3.10 или новее обычно подойдёт).

### Как проверить

1. Нажмите **Win**, введите **`cmd`** или **`PowerShell`**, откройте **Терминал** или **Windows PowerShell**.
2. Введите команду (и нажмите Enter):

```text
python --version
```

Если видите что-то вроде `Python 3.12.x` — хорошо.

Если пишет, что команда не найдена — установите Python с официального сайта [https://www.python.org/downloads/](https://www.python.org/downloads/) и при установке **включите галочку** «Add python.exe to PATH».

---

## 3. Где лежит проект

Дальше в примерах папка проекта — **`D:\project1`**. У вас путь может быть другой; главное — это та папка, где лежат файлы **`app.py`**, **`desktop.py`**, папка **`docs`** и т.д.

В терминале нужно **перейти в эту папку** перед установкой пакетов и запуском.

### Как перейти в папку в PowerShell

```powershell
cd D:\project1
```

(Замените `D:\project1` на ваш путь, если он другой.)

---

## 4. Один раз: установить зависимости

«Зависимости» — это дополнительные библиотеки, без которых программа не запустится.

Убедитесь, что вы в папке проекта (`cd` как выше), затем выполните:

```powershell
pip install -r requirements.txt
```

Это нужно для **основного** приложения и скриптов.

Если вы хотите запускать **окно с плеером** (рекомендуемый вариант для обычного использования), установите ещё:

```powershell
pip install -r requirements-desktop.txt
```

Дождитесь окончания (может занять минуту).

---

## 5. Как запустить Sapphire (два способа)

### Способ A — с окном (удобнее для вас)

1. Откройте **PowerShell** или **Терминал**.
2. Перейдите в папку проекта:

```powershell
cd D:\project1
```

3. Запустите:

```powershell
python desktop.py
```

Что произойдёт:

- В фоне запустится сервер на **`http://127.0.0.1:5000`**.
- Откроется **отдельное окно** с сайтом Sapphire (это делает программа **pywebview**).

**Важно:** не закрывайте терминал, пока пользуетесь плеером (или закрывайте — тогда сервер остановится). Окно терминала можно свернуть.

### Способ B — только браузер (без отдельного окна)

1. В той же папке проекта:

```powershell
cd D:\project1
python app.py
```

2. В терминале появится сообщение вроде `Running on http://127.0.0.1:5000`.
3. Откройте **браузер** (Chrome, Edge, Firefox) и в адресной строке введите:

```text
http://127.0.0.1:5000
```

Режим **`app.py`** удобен для разработки (есть отладка). Для обычного прослушивания чаще используют **`desktop.py`**.

## 6. Первый вход (логин и пароль)

При **первом** запуске, если в базе ещё нет пользователей, программа **сама** создаёт учётную запись:

| Поле     | Значение по умолчанию |
|----------|------------------------|
| Логин    | **`admin`**            |
| Пароль   | **`admin`**            |

На странице входа введите эти данные.

**Безопасность:** если компьютер доступен другим или сервер когда-нибудь будет в сети — смените пароль (через интерфейс, если есть, или задав переменную окружения до первого запуска — см. раздел про переменные в [CLOUD-UPLOAD-RU.md](CLOUD-UPLOAD-RU.md)).

---

## 7. Что открывать в браузере (кратко)

| Ситуация                         | Что ввести в адресной строке   |
|----------------------------------|---------------------------------|
| Сервер на этом же ПК, порт 5000  | `http://127.0.0.1:5000`        |
| Проверка, что сервер жив         | `http://127.0.0.1:5000/health-image` — должен вернуться JSON со `status: ok` |

С другого устройства в домашней сети адрес будет другой (IP вашего ПК), и сервер по умолчанию **не слушает внешние подключения** — для этого нужны отдельные настройки. Для начала достаточно **`127.0.0.1`**.

---

## 8. Загрузка музыки через сайт

После входа используйте элементы интерфейса плеера для загрузки файлов (если в вашей версии есть кнопка загрузки — она отправляет файл на тот же сервер).

Поддерживаются обычно форматы **`.mp3`** и **`.wav`** (лимит размера настраивается в коде, по умолчанию порядка 25 МБ).

---

## 9. Загрузка через скрипт (если нужно без браузера)

Когда сервер **уже запущен** (`desktop.py` или `app.py`), в **другом** окне терминала:

```powershell
cd D:\project1
python scripts\sapphire_upload.py -f "D:\Музыка\трек.mp3" -p admin
```

- **`-f`** — полный путь к файлу в кавычках, если в пути есть пробелы.
- **`-p`** — пароль (по умолчанию логин `admin`).

Скрипт сам залогинится и отправит файл; вручную копировать «сессию» не нужно.

---

## 10. Как правильно «выключить»

- Закройте окно **pywebview** или нажмите **Ctrl+C** в терминале, где запущен **`python app.py`** или **`python desktop.py`**.
- После этого адрес `http://127.0.0.1:5000` перестанет открываться — это нормально.

## 11. Частые проблемы

### «Не удаётся получить доступ к сайту» / страница не открывается

- Сервер **не запущен** — снова выполните `python desktop.py` или `python app.py`.
- Неверный адрес — должно быть именно **`http://127.0.0.1:5000`** (не `https` для локального запуска по умолчанию).

### «Порт 5000 уже занят»

Другая программа использует порт **5000**. Закройте другой запуск Sapphire или другое приложение на этом порту, либо измените порт в коде (это уже для продвинутых пользователей).

### `ModuleNotFoundError: No module named 'flask'`

Вы не установили зависимости или используете «другой» Python. Выполните из папки проекта:

```powershell
pip install -r requirements.txt
```

и снова `python desktop.py`.

### Скрипт загрузки пишет про `requests`

```powershell
pip install requests
```

(или снова `pip install -r requirements.txt` — там уже есть `requests`.)

### Забыли пароль от `admin`

Если это только локальная тестовая база, можно удалить файл базы **`instance\sapphire.db`** (в папке проекта, подпапка `instance`) и **запустить приложение снова** — создастся новый `admin` / `admin`. **Все загруженные треки и плейлисты в этой базе пропадут.** Делайте так только если понимаете последствия.

---

## 12. Нужен сайт в интернете «как стриминг», а не только на своём ПК

Локальный запуск (`127.0.0.1`, `desktop.py`) — это **для одного компьютера**. Чтобы открыть плеер **по домену из браузера у любого пользователя**, нужен **сервер в сети**, HTTPS, база данных и т.д.

Важно: это **ваш** сервис с **вашими загруженными треками**, а не каталог лицензированной музыки уровня Яндекс.Музыки. Подробно: **[ДЕПЛОЙ-В-ИНТЕРНЕТЕ-RU.md](ДЕПЛОЙ-В-ИНТЕРНЕТЕ-RU.md)**.
